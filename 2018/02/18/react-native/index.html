  4. 然后需要安装ios和android依赖

安装ios依赖

安装或更新xcode即可

安装android依赖

安卓依赖需要较多步骤，可参考官方文档。
主要分为三步：

  1. 安装SDK
  • 安装最新版本JDK(java开发工具包)：JDK。
  • 安装Android SDK：通过brew install android-sdk 安装。
  • 在shell配置文件中(~/.bashrc、~/.zshrc、或其他shell)正确导出ANDROID_HOME环境变量: export ANDROID_HOME=/usr/local/opt/android-skd
  • 执行source命令使得配置可以立即生效
  • 命令行执行android命令，打开Android SDK管理器。除默认选中之外，确保选中Android SDK Build-tools version 23.0.1、Android 6.0(API 23)、Android Support Repository，进行安装
  1. 安装模拟器工具
    运行android命令启动Android SDK管理器。安装
  • Intel x86 Atom System Image(for Android 5.1.1-API 22)
  • Intel X86 Emulator Accelerator(HAXM installer)
  1. 创建模拟器
    运行android avd命令启动AVD管理器创建管理器，记得勾选Use Host GPU选项

部署App到设备

部署到ios设备

  1. 需要一个Apple开发者帐号，在xcode登录
  2. 为帐号生成证书。打开通用面板(General)，点击Fix Issue来一步步获取证书。
  3. 登录到Apple开发者中心(http://developer.apple.com)，注册设备。
    其中uuid，可将设备连接到电脑，打开iTunes，选择设备，点击序列号，可看到。
    另外，部署前，需要在AppDelegate.m文件中奖localhost改成Mac的IP，然后就可以在xcode中点项目然后部署到设备了。

# 基本组件
react native中，不能使用html中的如div, p, img等标签了，只能使用native自己定义的标签。

View

容器组件。相当于html中的div

Text

文本组件。只能通过Text来包含文本。其他标签不能直接包含文本。

TextInput

文本输入组件。

onSubmitEditing

输入框内容更新。

Image

图片组件。

区分分辨率图片组件

比如通过xcode导入几倍图，然后用这个方式导入，可自动区分使用几倍图。这种图片需要通过xcode导入。

普通图片组件

普通web图片资源。这种，就不是App的本地图片了，属于每次都要请求图片资源。

ImageBackground

背景图片。

resizeMode

  1. contain
  2. cover
  3. stretch
    拉伸来铺满

ListView

用于显示垂直的滚动列表。ListView并不立即渲染所有元素，会优先渲染屏幕可见元素。
ListView必须的两个属性是dataSource和renderRow。
其中dataSource为数据源，renderRow为单个组件。

Navigator

起到路由器作用。

使用Stylesheet.Create

Stylesheet.Creat是可选的，但一般都会用它。
保证了值不可变；转成指向内部表的纯数字，保持了代码不透明性；将它放在文件末尾，可保证在应用中只会创建一次

触摸

触摸可通过<TouchableHighlight>把把界面元素包起来。会产生一个叠层，给用户视觉反馈。并且提供了一些钩子函数。
