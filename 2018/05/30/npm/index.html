pkg.main 和 pkg.module 说明

  1. pkg.main：定义了引入路径。不支持module字段时import引入包名时的路径。约定此处代码是基于ES5规范
  2. pkg.module：定义了引入路径。支持module字段时import引入的路径。此处代码是基于ES6规范的使用ES5语法的模块。为的是享受Tree Shaking
    Tree Shaking代表打包的代码会屏蔽项目中未使用的api，配合uglifyjs，能移除不用的代码。
    构建满足module字段方法：如用Rollup，设置output

添加单元测试

添加单元测试，可让项目获取build passing标志，是证明可用性的证书。
单元测试是在node环境跑的，因此需要让node认识window。可通过jsdom

然后可运行yarn test和yarn cover来看测试是否通过及测试覆盖率。
常见测试框架

  1. mocha：测试框架
  2. chai：断言库，判断代码结果对不对
  3. jsdom：node端是没有js dom对象的，比如window、document等等，所以需要这个库提供
  4. istanbul：代码覆盖率计算工具
  5. coveralls：统计代码测试覆盖率
  6. travis-ci：自动集成，如代码push到github后，travis-ci会自动进行测试
