防抖动和节流阀及raf的应用

发表于 2018-05-18   |   分类于 Javascript

防抖(debounce)和节流(throttle)都是用来控制某个函数在一定时间内执行多少次的技巧。
两者的主要区别为：
防抖为直到超过触发事件的时间间隔，才会执行，如果一直不超过，则会一直延迟下去
节流为触发事件过程中，定期执行事件

防抖，debounce

可使用lodash的_.debounce
debounce也可取消。

防抖适用场景

把触发频繁的事件，比如按键合并成一次执行。
如文本输入、自动完成的keyup事件

节流，throttle

可使用lodash的_.throttle

节流适用场景

如鼠标移动的mousemove事件
resize和scroll事件

requestAnimationFrame

跟_.throttle(dosomething, 16)等价。
不过追求更好的精确度的花，可使用raf

优点

动画保持60fps(每一帧16ms)，浏览器内部决定渲染最佳时机
标准的api

缺点

动画的开始取消需要自己控制，不像.debounce或.throttle由函数内部处理
浏览器标签未激活时，不会执行
IE9，Opera mini和老的android还是需要补丁

raf适用场景

只要涉及到重新计算元素位置，就可以使用
涉及ajax，添加/移除class，选择.debounce或者.throttle

#debounce #throttle #requestAnimationFrame
浏览器中四种不同类型的观察者
