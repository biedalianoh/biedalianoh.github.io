浏览器中四种不同类型的观察者

发表于 2018-05-17   |   分类于 Javascript

IntersectionObserver

用于观察两个DOM元素之间的交集。可用来检测DOM进入或离开可见视窗。

  1. 用来延迟加载图像或其他资源
  2. 识别广告可见性并计算广告收入
  3. 实现网站无限滚动
  4. 元素在视窗中时，加载和自动播放视频或动画

使用

兼容性

firefox和chrome支持。不支持的浏览器可使用对应的polyfill。

MutationObserver

用于观察DOM元素的变化。可用来检测添加或删除子节点、属性值变化、内容变化等。
MutationObserver被DOM3中的Mutation所替代。

使用

兼容性

兼容性良好
ie11, edge16, firefox58, chrome64, safari11支持

ResizeObserver

用于观察DOM元素内容宽高的变化。可用来检测元素大小变化。

  1. 元素被插入或删除，会触发
  2. display值为none时，会触发
  3. 元素显示时，且大小不是0，0，会触发
  4. 内联元素替换时，会触发
  5. 内联元素大小变化时，不会触发
  6. css的transform，不会触发

使用

兼容性

目前仅有chrome64+支持。大多数不支持

PerformanceObserver

用于观察性能时间轴。用来度量浏览器和Node.js某些性能指标。
浏览器：window.PerformanceObserver
Nodejs: const {performance} = require(‘perf_hooks’)

  1. 浏览器，发送请求和响应之间的处理时间。
  2. Nodejs，从数据库中检索数据计算持续时间。
  3. 抽象精确的时间信息，使用Paint Timing API，比如First Paint或First Contentful Paint时间
  4. 使用’User Timing API’、’Navigation Timing API’、’Network Information API’、’Resource Timing API’、’Paint Timing API’，访问性能指标。
