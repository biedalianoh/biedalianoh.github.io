Blob对象应用，获取二进制对象及使用实例

发表于 2018-07-27   |   分类于 Javascript

在计算机中，Blob通常是数据库中用来存储二进制文件的字段类型。
在HTML5中，Blob是一种js的对象类型，可用来存储二进制数据。一个Blob对象是一个包含只读原始数据的类文件对象。
js中的File对象，就是继承自Blob对象。

基本信息

创建好的Blob对象，是只读的。

创建

第一个参数为一个数据序列，可以是任意格式的值
第二个参数为一个包含两个属性的对象:
{
type: MIME类型,
endings: 决定第一个参数的数据格式 //可取值transparent(默认), native(按操作系统转换)
}

MIME

设定某种扩展名的文件用哪种应用程序来打开的类型。多用于指定一些客户端自定义的文件名，以及一些媒体文件打开方式。
常见MIME类型：

属性

  1. Blob.isClosed
    表明blob对象是否已经销毁。若已销毁则不可读。
  2. Blob.size
    所包含数据大小。
  3. Blob.type
    返回MIME类型。若未知，则返回空字符串

方法

  1. Blob.close()
    销毁Blob对象，释放底层资源。
  2. Blob.slice([start[,end[,contentType]]])
    对这个blob对象的数据进行切割，返回一个新的Blob对象，包含源Blob对象中指定范围的数据。
    对文件分片上传的时候使用。

实现

读取文件

读取文件，主要是用到继承自Blob的File对象。
File对象本身只包含文件名，文件类型等只读属性。唯一能读取File对象文件内容的途径是使用FileReader构造函数。

clipboardData

继承自DataTransfer对象。

属性

  1. dropEffect 默认为none
  2. effectAllowed 默认为uninitialized
  3. files 粘贴操作为空List
  4. items 剪切板中的各项数据
  5. types 剪切板中的数据类型，该属性在safari中比较混乱

items

items为一个DataTransferItemList实例。
items的每一项item有两个属性kind和type。

  1. kind
    一般为’string’或’file’
  2. type
    具体数据类型。哪种类型字符串(MIME-Type)或哪种文件类型。
    type值有:

FileReader

用来读取文件file或blob，获取文件内容。

属性

readyState

表示FileReader状态。
0–还没加载任何数据
1–数据正在被加载
3–已完成全部读取请求

result

通过对应读取方式获取到的结果

error

触发onerror事件时，错误信息会保存在error属性的对象中。这个对象只有一个属性code：
1–表示未找到文件
2–表示安全性错误
3–表示读取中断
4–表示文件不可读
5–表示编码错误

readAsArrayBuffer(file)

将文件读取为ArrayBuffer对象
按字节byte读取文件二进制内容，并转换为ArrayBuffer对象。读取后不会进行编码与源文件大小一样，这是和readAsBinaryString的主要区别。
（ArrayBuffer是存放了一段二进制数据的内存空间）
（展现出来的数据，使用了类型化数组，数组中的每一个元素是当前字节的十进制数值）
