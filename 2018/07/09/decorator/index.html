修饰器函数@修改类和类的方法

发表于 2018-07-09   |   分类于 Javascript

修饰器(Decorator)函数用来改变类的行为。用来给类添加方法和属性，给类的方法修改描述。

基础使用

修饰器的使用，只能用于类和类的方法。
不能用于函数，因为存在函数提升。

完整参数

修饰器函数一共可以接受三个参数，第一个是所要修饰的目标对象，第二个是所要修饰的属性名，第三个是该属性的描述对象(即对象的描述如writeable之类的)。
如果修饰类，一般只需要一个参数target，如果是修饰类的属性，则使用三个参数。

修饰器-修饰类的属性

当修饰属性时，第一个参数是类的原型对象prototype；第二个是要修饰的属性名，第三个是该属性的描述对象。
描述对象descriptor为：
{
value: specifiedFunction,
enumerable: false,
configurable: true,
writable: true
}

多个修饰器

一个方法可以使用多个修饰器。
当使用多个修饰器时，会从外进入到内，然后由内向外执行。

综合使用

良好的使用修饰器函数，能使组件的属性和功能一目了然。
@testable
@Component({
tag: ‘my-component’,
styleUrl: ‘my-compoent.scss’
})
export class MyCompoent {
@Prop() first: string;
@Prop() last: string;
@State() isVisible: boolean = true;

@readonly
@nonenumerable
name() {return `${name}`}

render() {
    return (
        <p>Hello, my name is {this.first} {this.last}</p>
    );
}

}

第三方修饰器工具

core-decorators.js

core-decorators提供了几个常见的修饰器。

@readonly

使属性或方法不可写

@deprecate / deprecated

在控制台显示一条警告，表示该方法将废除。

traits-decorators.js

traits效果与Mixins效果类似，但是功能更多。
traits修饰器，不仅可以接受对象，还可以接受 ES6 类作为参数。
