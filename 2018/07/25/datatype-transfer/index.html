Javascript强制数据类型转换

发表于 2018-07-25   |   分类于 Javascript

显示数据类型转换

数值类型

Number(), parseInt(), parseFloat()
通过Number的实例来说明，其他2个类似Number。

非对象转换

对象转换

转换的是对象时：先调用valueOf()方法，然后依据非对象转换的规则转换返回的值；转换后如果是NaN，再调用toString()，然后再按非对象转换一次。

字符串类型

String()
可将任何类型转为字符串。

  1. 调用时，如果有toString()方法，则直接调用该方法。null和undefined没有这个方法。其他都有
  2. null返回’null’
  3. undefined返回’undefined’

布尔类型

以下值会被转为false: false, ‘’, 0, NaN, null, undefined
其他都会转为true

隐式转换

某些情况，js会自动进行类型转换。

isNaN

调用isNaN时，js会自动调用Number()进行转换。
获取到Number()返回值后，如果是非数值，则返回true，如果是数值，则返回false

递增、递减操作符, 一元正负操作符

会先对变量执行Number()操作。若转化为正常数字类型，则执行操作。
若返回不是正常数字类型，则返回NaN

加法操作符

若其中有一个为字符串，则将另一个通过String()转为字符串，然后连接起来。
若均为非字符串，则调用Number()转化。

减法、乘除、取模操作符

所有非数值的变量均会调用Number()进行转换。

逻辑操作符(!、&&、||)

均通过Boolean()转换变量，然后判断

大小操作符(<、>、<=、>=)

任意类型，都可进行大小操作符判断。

  1. 均为数值，则直接比较大小
  2. 均为字符串，则比较字符串对应的字符编码值
  3. 如有一个为数值，则将另一个转为数值，进行比较。若另一个为NaN，则肯定为false
  4. 如有一个为对象，则调用valueOf()方法，若没有此方法，则调用toString()方法，然后按之前规则比较。由于对象为NaN，所以肯定为false
  5. 如有变量为布尔值，则转为数值进行比较
  6. 如有一个为NaN，则肯定返回false

字符串编码

字符串编码发展的历史：

  1. 最先提出ASCII码，用于在不同硬件软件中实现数据传输标准化。目前属于国际标准。由7或8为二进制数字来表示。如1100001表示A
  2. 后来各国为显示本国语言，各自指定编码标准。如大陆GB2312，台湾BIG5，日本JIS
  3. 插一个，由于很多大陆GB2312不能编码很多生僻字，后来出现了GBK
  4. 由于存在各类编码造成混乱，所以出现了Unicode编码，各类编码都能切换到Unicode，也能再切换回去。Unicode的缺点是存储和传输很耗资源。
  5. 为了提高Unicode编码效率，出现了UTF-8。实际过程为，如”汉”字：汉的Unicode编码是U+00006C49，把U+00006C49通过UTF-8编码器进行编码，输出E6B189
  6. 有的电子邮件需要使用Base64编码。如发送时GB2312–>根据Base64规则–>转换为ASCII码，接收时反过来。

相等操作符(==)

任意类型，都可进行相等操作符判断。

  1. 如有一个为布尔值，则转为数值
  2. 如有一个为字符串，另一个为数值，则用Number()转换字符串为数值
  3. 如有一个为对象，另一个不是，则调用对象的valueOf(), 然后按之前规则比较。由于对象为NaN，则肯定为false
  4. null与undefined是相等的
  5. 如有一个为NaN，则肯定返回false
  6. 如两个均为对象，则比较指向是否相同

总结

特别是在使用isNaN、加减乘除、对比操作符时，会进行隐式转换，这个转换主要为：普通变量用Number()转换，对象通过valueOf()和toString()方法转过之后然后再Number()转换。
然后记住：

  1. boolean值转为1或0
  2. 两个字符串做对比，会比较字符串编码值
  3. 其他
