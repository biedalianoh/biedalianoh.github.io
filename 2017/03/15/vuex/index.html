vuex项目应用/开发实战

发表于 2017-03-15   |   分类于 Javascript

Vuex 是一个专为 Vue.js 应用程序开发的状态管理模式。它采用集中式存储管理应用的所有组件的状态，并以相应的规则保证状态以一种可预测的方式发生变化。
Vuex 也集成到 Vue 的官方调试工具 devtools extension，提供了诸如零配置的 time-travel 调试、状态快照导入导出等高级调试功能。
Vuex的应用状态分为state(数据源), view(视图), actions(状态变化)
通常复杂的单页应用，需要父子，多层父子，兄弟组件之间传递状态，应用vuex

vuex

Vuex 中 Store 的模板化定义如下：

state

state定义了应用状态的数据结构，同样可以在这里设置默认的初始状态。

actions

actions是定义提交触发更改信息的描述，常见的有从服务端获取数据，获取数据后调用store.commit()来调用更改store中的状态。
可以在组件中使用dispatch来发出actions

mutations

mutations是唯一允许更新应用状态的地方。

getters

getters允许组件从store中获取数据

modules

modules对象允许将单一的store拆分为多个store同时保存在单一的状态树中。随着应用复杂度的增加，这种拆分能更好地组织代码。
