Anroid开发入门

发表于 2017-02-17   |   分类于 Android

相比在学习ios开发时的举步维艰，android开发学习则顺畅很多，在粗略看完Android官方培训课程中文版(v0.9.7)之后，开始看Android第一行代码，并做笔记。

Activity

活动。是可以包含用户界面的组件。用户界面需要通过活动来展现。

创建活动

  1. 右键java包–New–Activity–Empty Activity
    Generate Layout File: 创建一个对应的布局文件
    Launcher Activity: 设置为主活动
  2. 在新建活动java文件中重写onCreate方法
    AS自动帮忙实现了@Override
  3. 创建对应Layout布局文件
    右键Layout目录–Layout resource file
  4. 在.xml布局文件中添加元素
    添加Button, text等元素
  5. 加载布局
    在活动java文件中添加布局：setContentView(R.layout.myAddLayout)
  6. 在menifest中注册活动
    AS自动帮忙注册了

启动活动

Intent是各组件之间进行交互的一种重要方式。
可指定当前组件想要执行的动作，还可在不同组件间传递数据
Intent可用于启动活动，启动服务，发送广播等场景。

显式Intent

Intent接收两个参数：启动活动的上下文，目标活动

隐式Intent

不明确指出启动哪一个活动，而是指出一系列抽象的action和category等信息，然后由系统去分析应该启动哪一个活动。
action和category是在activity中的intent-filter里设置的。

如果menifest中的activity有相应的action和category，则打开对应的activity。

  2. 可在intent-filter中配置标签，用来指定活动能响应的数据类型

    android:port 端口。如8080
    android:path 路径。如/kylin/a.html
    android:mimeType 数据类型。

传递活动数据

发送数据

使用putExtra()方法传递数据。

接收数据

  1. 通过getIntent()获取启动当前活动的活动。

  2. 通过getStringExtra()方法获取传递的字符串数据

还有其他类型的方法：
getIntExtra() 获取整形数据
getBooleanExtra() 获取布尔值数据

返回数据

  1. 使用startActivityForResult()启动活动
    startActivityForResult()方法用于启动活动，不过在活动销毁的时候能返回一个结果给上一个活动。
    第二个参数为启动活动的标志符

  2. 使用setResult设置回传数据
    第一个参数有两种值：RESULT_OK, RESULT_CANCELED
    第二个参数为活动

  3. 覆写onActivityResult()方法获取数据
    第一个参数为唯一标志符
    第二个参数为处理结果
    第三个参数为活动

  4. 覆写onBackPressed()方法设置回传数据
    若是通过Back按钮回到上一个活动，则需要重写onBackPressed()方法来设置回传数据。

活动启动模式

设置活动在栈中储存的情况。分为4种模式。
通过设置menifest文件中的launchMode来实现。

standard

默认模式。即每次调用活动都会在栈中添加一个新的activity。

singleTop

当前活动调用当前活动，则不会创建新的活动。

singleTask

调用活动时，查找栈中是否有此活动，有则调用，没有则创建。

singleInstance

其他程序调用此活动时，创建一个新的活动栈。和其他程序共享此活动程序。
