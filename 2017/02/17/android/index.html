Anroid开发入门

发表于 2017-02-17   |   分类于 Android

相比在学习ios开发时的举步维艰，android开发学习则顺畅很多，在粗略看完Android官方培训课程中文版(v0.9.7)之后，开始看Android第一行代码，并做笔记。

Activity

活动。是可以包含用户界面的组件。用户界面需要通过活动来展现。

创建活动

  1. 右键java包–New–Activity–Empty Activity
    Generate Layout File: 创建一个对应的布局文件
    Launcher Activity: 设置为主活动
  2. 在新建活动java文件中重写onCreate方法
    AS自动帮忙实现了@Override
  3. 创建对应Layout布局文件
    右键Layout目录–Layout resource file
  4. 在.xml布局文件中添加元素
    添加Button, text等元素
  5. 加载布局
    在活动java文件中添加布局：setContentView(R.layout.myAddLayout)
  6. 在menifest中注册活动
    AS自动帮忙注册了

启动活动

Intent是各组件之间进行交互的一种重要方式。
可指定当前组件想要执行的动作，还可在不同组件间传递数据
Intent可用于启动活动，启动服务，发送广播等场景。

显式Intent

Intent接收两个参数：启动活动的上下文，目标活动

指定之后，通过startActivity()方法就可启动目标活动。如：

隐式Intent

不明确指出启动哪一个活动，而是指出一系列抽象的action和category等信息，然后由系统去分析应该启动哪一个活动。
action和category是在activity中的intent-filter里设置的。
指定action和category方法：

如果menifest中的activity有相应的action和category，则打开对应的activity。

隐式Intent扩展

  1. 可使用隐式Intent去调用其他程序的活动。
    如调用浏览器：
    其中Intent.ACTION_VIEW为系统内置动作，通过从Uri.parse()方法将一个网址解析成Uri对象，再用setData()将数据传递进去。

  2. 可在intent-filter中配置标签，用来指定活动能响应的数据类型
    中可配置内容：
    android:scheme 协议。如http
    android:host 主机名。如www.baidu.com
    android:port 端口。如8080
    android:path 路径。如/kylin/a.html
    android:mimeType 数据类型。

  3. 调用拨号

传递活动数据

发送数据

使用putExtra()方法传递数据。

接收数据

  1. 通过getIntent()获取启动当前活动的活动。

  2. 通过getStringExtra()方法获取传递的字符串数据

还有其他类型的方法：
getIntExtra() 获取整形数据
getBooleanExtra() 获取布尔值数据

返回数据

  1. 使用startActivityForResult()启动活动
    startActivityForResult()方法用于启动活动，不过在活动销毁的时候能返回一个结果给上一个活动。
    第二个参数为启动活动的标志符

  2. 使用setResult设置回传数据
    第一个参数有两种值：RESULT_OK, RESULT_CANCELED
    第二个参数为活动

  3. 覆写onActivityResult()方法获取数据
    第一个参数为唯一标志符
    第二个参数为处理结果
    第三个参数为活动

  4. 覆写onBackPressed()方法设置回传数据
    若是通过Back按钮回到上一个活动，则需要重写onBackPressed()方法来设置回传数据。

活动启动模式

设置活动在栈中储存的情况。分为4种模式。
通过设置menifest文件中的launchMode来实现。

standard

默认模式。即每次调用活动都会在栈中添加一个新的activity。

singleTop

当前活动调用当前活动，则不会创建新的活动。

singleTask

调用活动时，查找栈中是否有此活动，有则调用，没有则创建。

singleInstance

其他程序调用此活动时，创建一个新的活动栈。和其他程序共享此活动程序。

活动相关技巧

知晓当前是哪一个活动

  1. 建立一个通用基础类
    新建一个java类，并让所有活动都继承这个类。

退出程序

正常退出程序需要按很多次Back键，可专门用一个类来对所有的活动进行管理。

封装启动活动方法

特别是需要给下一个活动传递参数时，最好通过封装启动方法，这样所需传递参数一目了然。

UI

android自身提供了大量的UI控件。如TextView, Button。
它们的基本属性都类似。

TextView

layout_width, layout_height

  1. match_parent 和父元素大小相同
  2. fill_parent 和match_parent功能相同，官方推荐match_parent
  3. wrap_content 根据自身内容自适应
  4. 自定义固定大小

    gravity 对齐方式

    用于文字在控件中的对齐方式。
    可使用 top, bottom, left, right, center, center_vertical, center_horizontal。
    也可用 | 来同时指定多个值。

textSize

文字大小。android中使用sp作为单位。

textColor

文字颜色。

Button

基本属性和TextView一样。

textAllCaps

系统默认会把Button中所有英文字母自动转为大写。
可使用textAllCaps禁用。

EditText

输入框。

hint 占位符，类似placeholder

maxLines

最大行数。超过这个行数，则自动滚动。

getText()

获取输入内容。

ImageView

图片。
其中图片资源文件夹有drawable, drawable-hdpi等，和mipmap, mipmap-hdpi等。
官方推荐用mipmap。
导入图片后，直接生成了相应变量。

setImageResource()改变图片路径

ProgressBar

进度条。默认为圆形loading图

getVisibility()

UI组件通用方法。
此方法为获取显示状态。
值可为：
visible, 可见
invisible, 不可见，占据位置
gone, 不可见，不占据位置

setVisibility()

水平进度条

通过设置样式，可设置成水平进度条

getProgress()

获取进度

setProgress()

设置进度

AlertDialog

弹出置顶对话框。类似confirm

定义

setTitle()

setMessage()

setCancelable()

是否能用返回键退出。
默认为true，能用返回键退出。

setPositiveButton()

设置确定按钮点击事件。

setNegativeButton()

设置取消按钮点击事件。

ProgressDialog

类似AlertDialog，不同的是会在对话框中显示一个进度条。

dismiss()

关闭对话框。

自定义控件

首先要创建自定义控件的布局，然后使用include引入布局，然后把需要替代的控件隐藏。
若要添加自定义控件的方法，并且不想每个活动都去写这个方法，则不使用include，而是自定义布局类。

include

引入文件。

getSupportActionBar()

获取自带标题栏。

自定义布局类

定义好布局类之后，可直接使用布局类来添加控件。像Button, EditText之类一样使用。

同时，没有使用layout_weight的控件，将自适应，其它使用了的控件将按剩余宽度进行栅格化。

RelativeLayout相对布局

直接指定控件的位置。

layout_alignParentLeft

固定在左。

layout_alignParentRight

固定在右。

layout_alignParentTop

固定在顶部。

layout_alignParentBottom

固定在底部。

layout_centerInParent

固定在页面中间。

layout_above

在控件上方。
注意：在引用id时，需要该id的控件已经声明。

layout_below

在控件下方

layout_toLeftOf

在控件左方

layout_toRightOf

在控件右方
