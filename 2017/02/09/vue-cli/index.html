目录结构

build

build目录为运行不同环境时所使用的webpack命令

config

config目录为运行不同环境时的配置文件

dist

为运行线上环境所生成的代码，线上访问时访问的文件。

src

项目开发的主目录。

assets

存放图片，音视频等资源文件

components

存放vue各种组件

static

静态资源。即编译时原样输出的文件。

test

运行测试的文件。

搭建路由

搭建vue路由主要通过vue-router包来实现。可分为路由实现和路由配置文件两个文件。

路由实现

路由中主要的部分为routes，即路径。其他额外选项功能可查看vue-router

路由配置

路由配置文件，需要定义响应路径path，和这个path下对应的component。
component内容可以单独使用文件引入，也可直接在配置文件中定义。

页面开发

项目组成为，使用main.js为总成文件，引入各类框架，如element，vue-router，并定义了总入口的vue文件，和绑定的dom元素。

vue页面组成

一个vue页面中，可以包含三种类型元素：template，script，style，分别代表了html,js,css

vue页面语法

配置选项

新建一个vue对象的全部配置选项：

全局api

Vue.extend

创建一个包含Vue组件选项的对象

Vue.nextTick

在下次 DOM 更新结束之后执行函数

Vue.set

Vue 不允许在已经创建的实例上动态添加新的根级响应式属性。然而它可以使用 Vue.set(object, key, value) 方法将响应属性添加到嵌套的对象上

Vue.delete

删除对象的属性。

Vue.directive

注册或获取全局指令。注册自己的类似于v-show，v-model指令。

Vue.filter

注册或获取全局过滤器。

Vue.component

注册或获取全局组件。注册还会自动使用给定的id设置组件的名称

Vue.use

安装 Vue.js 插件。

Vue.mixin

全局注册一个混合，影响注册之后所有创建的每个 Vue 实例。插件作者可以使用混合，向组件注入自定义的行为。不推荐在应用代码中使用。

Vue.compile

在render函数中编译模板字符串。只在独立构建时有效

实例属性

vm.$data

Vue 实例观察的数据对象。Vue 实例代理了对其 data 对象属性的访问。

vm.$el

Vue 实例使用的根 DOM 元素。

vm.$options

用于当前 Vue 实例的初始化选项。需要在选项中包含自定义属性时会有用处：

vm.$parent

父实例，如果当前实例有的话。

vm.$root

当前组件树的根 Vue 实例。如果当前实例没有父实例，此实例将会是其自已。

vm.$children

当前实例的直接子组件。需要注意 $children 并不保证顺序，也不是响应式的。如果你发现自己正在尝试使用 $children 来进行数据绑定，考虑使用一个数组配合 v-for 来生成子组件，并且使用 Array 作为真正的来源。

vm.$slots

用来访问被 slot 分发的内容。每个具名 slot 有其相应的属性（例如：slot=”foo” 中的内容将会在 vm.$slots.foo 中被找到）。default 属性包括了所有没有被包含在具名 slot 中的节点。
在使用 render 函数书写一个组件时，访问 vm.$slots 最有帮助。

vm.$scopedSlots

vm.$refs

一个对象，其中包含了所有拥有 ref 注册的子组件。

vm.$isServer

当前 Vue 实例是否运行于服务器。

实例方法

vm.$on

监听当前实例上的自定义事件。事件可以由vm.$emit触发。回调函数会接收所有传入事件触发函数的额外参数。

vm.$once

监听一个自定义事件，但是只触发一次，在第一次触发之移除除监听器。

vm.$off

移除事件监听器。

vm.$emit

触发当前实例上的事件。附加参数都会传给监听器回调。

vm.$watch

观察 Vue 实例变化的一个表达式或计算属性函数。回调函数得到的参数为新值和旧值。表达式只接受监督的键路径。对于更复杂的表达式，用一个函数取代。

vm.$watch 返回一个取消观察函数，用来停止触发回调：

vm.$set

这是全局 Vue.set 的别名。

vm.$delete

这是全局 Vue.delete 的别名。

DOM指令

v-text

更新元素的 textContent。

v-html

更新元素的 innerHTML 。

v-show

根据表达式之真假值，切换元素的 display CSS 属性。

v-if

根据表达式的值的真假条件渲染元素。

v-else

为 v-if 或者 v-else-if 添加 “else 块”。

v-else-if

为 v-if 添加 else if 块

v-for

基于源数据多次渲染元素或模板块。

v-on

绑定事件监听器。
.stop - 调用 event.stopPropagation()。
.prevent - 调用 event.preventDefault()。
.capture - 添加事件侦听器时使用 capture 模式。
.self - 只当事件是从侦听器绑定的元素本身触发时才触发回调。
.{keyCode | keyAlias} - 只当事件是从侦听器绑定的元素本身触发时才触发回调。
.native - 监听组件根元素的原生事件。

v-bind

动态地绑定一个或多个特性，或一个组件 prop 到表达式。
.prop - 被用于绑定 DOM 属性。
.camel - transform the kebab-case attribute name into camelCase. (supported since 2.1.0)

v-model

在表单控件或者组件上创建双向绑定。
.lazy - 取代 input 监听 change 事件
.number - 输入字符串转为数字
.trim - 输入首尾空格过滤

v-pre

跳过这个元素和它的子元素的编译过程。可以用来显示原始 Mustache 标签。跳过大量没有指令的节点会加快编译。

v-cloak

这个指令保持在元素上直到关联实例结束编译。和 CSS 规则如 [v-cloak] { display: none } 一起用时，这个指令可以隐藏未编译的 Mustache 标签直到实例准备完毕。

v-once

只渲染元素和组件一次。随后的重新渲染,元素/组件及其所有的子节点将被视为静态内容并跳过。这可以用于优化更新性能。

key

使用key，它会基于key的变化重新排列元素顺序，并且会移除key不存在的元素。
最常见的用例是结合 v-for:

ref

ref 被用来给元素或子组件注册引用信息。引用信息会根据父组件的 $refs 对象进行注册。如果在普通的DOM元素上使用，引用信息就是元素; 如果用在子组件上，引用信息就是组件实例:

slot

用于标记往哪个slot中插入子组件内容。

组件引用

项目入口处，引用vue总组件

除了项目入口之外，组件可直接在template的语法中引用。
并且js中为驼峰写法，html中用-连接

组件间数据传递

  1. 父组件传数据给子组件
    使用props传递数据

  2. 子组件传数据给父组件
    通过自定义事件来触发状态。
    父组件上使用v-on方法来监听一个事件，子组件中使用$emit()来触发这个事件

  3. 父组件访问子组件
    尽管有props，但有时仍想在js中之杰访问子组件。可使用v-ref指定索引ID后访问。

  4. 父组件传递数据覆盖子组件默认数据
    通过slot传递父组件数据

  5. 传递数据验证
    验证props，此时props为一个json对象
