Avalon

兼容IE6.
国内出的工具，使用vbs解决了defineProperty在IE6,7,8的兼容问题。

工具类

lodash

lodash 从4.0.0开始支持的环境有： Chrome 46-47, Firefox 42-43, IE 9-11, Edge 13, Safari 8-9, Node.js 0.10.x, 0.12.x, 4.x, & 5.x, & PhantomJS 1.9.8。已不再支持IE6~IE8。
如果想兼容IE6~IE8，可以使用3.x版本。3.x版本支持的环境有：
Chrome 43-44, Firefox 38-39, IE 6-11, MS Edge, Safari 5-8, ChakraNode 0.12.2, io.js 2.5.0, Node.js 0.8.28, 0.10.40, & 0.12.7, PhantomJS 1.9.8, RingoJS 0.11, & Rhino1.7.6
lodash 3.x 版本没有直接提供可用的js， 需要手动构建。

安装

swiper

兼容IE8，需要使用旧版：idangerous.swiper2.7.6.css 和 idangerous.swiper2.7.6.js
兼容IE7待测，可能需要包含jquery

fullpage

兼容IE8
fullpage与wow.js(滚动条滑到某个位置，才显示动画)的兼容:

主要起作用的代码是要把scrollBar设置为true,并且把动画事件放在afterRender里面。

jQuery

兼容IE9以下版本，需要使用jQuery-1.xx版本，经实际使用，jQuery v1.11.3 版本没有问题。
为防止jquery与其他框架强夺 $ 标志符，要执行jQuery.noConflict()

webpack

webpack在使用babel解析打包时，输出的为ES5版本。

兼容性

ES5 不支持 IE6/7，8支持一点，IE9支持更多，IE10完美支持，IE11支持少量ES6

解决方法

添加以下步骤：

  1. webpack进行babel对ES6，7语法的转码

  2. webpack引用es3ify-loader解决es3语法兼容问题。

  3. 全局引用babel-polyfill, es5-shim/es5-sham, JSON的polyfill等
    polyfill:

es5-shim/es5-sham:

  1. 不要在代码种用Object.defineProperty设置访问起属性，若第三方包种有，找到，改之。

  2. 模块导出不能使用 export default，改为export{xxx}

  3. 模块引入使用 import {} from ‘xxx’

bootstrap3

此为bootstrap3兼容ie8，若要兼容IE6，7，搜索bsie(bootstrap2)。

  1. 使用html5文档声明。

  2. 加入meta标签

  3. 引入bootstrap文件

  4. 引入respond.js和html5.js

  5. 使用bootstrap的js时要引入1.x版本jquery
