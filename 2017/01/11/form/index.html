form表单元素以及提交方式

发表于 2017-01-11   |   分类于 Javascript

H5表单项标签，ajax，formaction提交方式，表单包含文件提交方式，多风格展示数据。

新表单项标签

email

这种文本框在移动版浏览器下，其键盘会根据文本框类型不同而显示相对应的键盘

search

该文本框和普通文本框唯一区别，在 Safari 和 Chrome 浏览器下，文本框的外观为圆角

url

这种文本框在移动版浏览器下，其键盘会根据文本框类型不同而显示相对应的键盘

tel

这种文本框在移动版浏览器下，其键盘会根据文本框类型不同而显示相对应的键盘

number

可以配合 min、max、及step属性使用

range

提供的是一种滑动输入方式，需要配合 min、max、range等属性的使用

color

image

只有当input的type为image时，可以直接设置width，height。

datetime

日期和时间(含时区)。选取时间、日、月、年（UTC 时间）

datetime-local

日期和时间(不含时区)。选取时间、日、月、年（本地时间）

time

时间选择器文本框。选取时间（小时和分钟）

date

选取日、月、年

month

选取月、年

week

选取周和年

list, datalist

list用来提示文本框输入，提示的数据源由datalist提供。

progress

progress 标签定义运行中的进度（进程）。可以使用 progress 标签来显示 JavaScript 中耗费时间的函数的进度。

meter

标签定义度量衡。仅用于已知最大和最小值的度量。

output

output 标签定义不同类型的输出，比如脚本的输出。是一个数据的展示地。

keygen

keygen 标签规定用于表单的密钥对生成器字段。当提交表单时，私钥存储在本地，公钥发送到服务器。所有主流浏览器都支持 keygen 标签，除了 Internet Explorer 和 Safari。

表单项属性

accept-charset

action

autocomplete

表单自动完成。联想填过的属性。

autofocus

自动聚焦。

form

规定输入域所属的表单。添加form属性，外部的input就属于表单的提交范围.

formaction

重新指定表单的action属性。可以让表单的内容通过不同的提交按钮提交到不同的地方。

enctype

enctype 属性可能的值：
application/x-www-form-urlencoded
multipart/form-data
text/plain

methods

step

step规定输入框中输入内容的合法字符的步进数。

multiple

规定输入域中是否可选多个值。如加此属性后file可选多个文件

named

novalidate

规定在表单提交时不验证form或input项

pattern

验证input输入内容是否匹配正则表达式。

target

新表单方法

H5自带表单验证

当表单有类似required，pattern等对输入框有限制的条件时，submit提交时会验证并给出响应提示。

提示框内的文字可以使用setCustomValidity方法来自定义。

invalid事件会再submit事件之前触发，而提交时会先验证所有表单元素是值是否有效。除了提交外还可以手动调用checkValidity方法来执行验证。

另外表单元素会有自己的validity属性，validity为一个表单元素所有匹配的信息

id和name选择器

js中如果id名称不和内置属性或全局变量重名的话，该名称自动成为window对象的属性，即可以直接使用id获取元素。

对于name属性来说，只有某些类型的html元素具有类似的方法:
比如：form,img,iframe,applet,embed,object等。
这些元素可以通过全局变量或者document的属性来访问特定name属性的元素；
如果相同name属性的有多个元素，则返回一个NodeList的类似只读数组的对象。

表单提交

submit

form标签中添加action和method，且有一个submit按钮，所提交的数据需要有name属性。

这种方式会进行页面跳转。

ajax

jQuery.ajax支持属性：

jsonp:
远程服务器上设法把数据装进js格式的文件里，供客户端调用和进一步处理

数据展示

webgl, three.js, d3.js
webgl是HTML5中的一种3D绘图标准，允许把JavaScript和OpenGL ES 2.0结合在一起。
three.js是以webgl为基础的库，封装了一些3D渲染需求中重要的工具方法与渲染循环。
D3.js是一个数据可视化的库，技术基础是SVG。
