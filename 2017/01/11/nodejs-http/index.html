nodejs深入系列--http

发表于 2017-01-11   |   分类于 Javascript

http.Agent

用于支持socket和Connection: keep-alive。

new http.Agent[options]

示例：

options

  1. keepAlive
    保持链接池。默认为false
  2. keepAliveMsecs
    发送TCP保持连接包的频率。当keepAlive为true时使用。默认为1000
  3. maxSockets
    每个host能连接的最大sockets数量。默认为Infinity
  4. maxFreeSockets
    空闲状态能打开的最大sockets数。当keepAlive为true时使用。默认为256

globalAgent

Agent有一个全局实例globalAgent被用在所有的HTTP客户端请求。

agent.createConnection(option[, callback])

和net.createConnection()用法相同。

agent.destroy()

摧毁所有正在使用的sockets

agent.freeSockets

当处于keepAlive状态时，显示所有空闲状态的sockets。不要修改。

agent.getName(options)

options可选择：

  1. host
  2. port
  3. localAddress

agent.maxFreeSockets

获取空闲状态能打开的最大sockets数。

agent.maxSockets

获取每个host能连接的最大sockets数量。

agent.requests

显示尚未分配给sokects的的请求队列。不要修改。

agent.sockets

显示正在被使用中的sockets列表。不要修改。

http.ClientRequest

使用http.request()时返回的实例。代表一个正在进行的请求。
http.ClientRequest实例有两个参数：
http.request(options, function(res) {})

  1. options
    设置host，port等

  2. function(res) {}

http.ClientRequest能响应以下方法：

abort

当客户端被abort时，会emit触发http.ClientRequest实例的abort方法。只有第一次的时候会触发。

aborted

当服务端abort时，会emit触发。

connect

服务端响应请求时每次都会触发此方法。如果没有去监听此方法，则客户端会关闭连接。
拥有response，socket，head参数。

continue

当服务端发送 ‘100 Continue’ 的http response时触发。这种是在请求中包含 ‘Expect: 100-continue’ 时会出现。

response

当响应请求时会触发此方法。此方法只会被触发一次。

socket

当一哥socket被分配到请求时触发。

upgrade

当服务端响应一个带有upgrade的请求时触发。若没有监听此方法，则带有upgrade header的请求会被关闭。

request.abort()

request.end([data][, encoding][, callback])

request.flushHeaders()

request.setNoDelay([noDelay])

request.setSocketKeepAlive([enable][, initialDelay])

request.setTimeout(timeout[, callback])

request.write(chunk[, encoding][, callback])

http.Server

http.Server继承自net.Server。并且添加了以下方法：

checkContinue

checkExpectation

clientError

close

connect

connection

request

upgrade

server.close([callback])

server.listen(handle[, callback])

server.listen(path[, callback])

server.listen([port][, hostname][, backlog][, callback])

server.listening

server.maxHeadersCount

server.setTimeout(msecs, callback)

server.timeout

http.ServerResponse

http.ServerResponse被http server所创建，当作request方法的第二个参数。
拥有以下方法：

close

finish

response.addTrailers(headers)

response.end([data][, encoding][, callback])

response.finished

response.getHeader(name)

response.headersSent

response.removeHeader(name)

response.sendDate

response.setHeader(name, value)

response.setTimeout(msecs, callback)

response.statusCode

response.statusMessage

response.write(chunk[, encoding][, callback])

response.writeContinue()

response.writeHead(statusCode[, statusMessage][, headers])

http.IncomingMessage

http.IncomingMessage被http.Server或http.ClientRequest所创建，当作request和response方法的第一个参数。
可用于访问reponse status, headers, data.
拥有以下方法：

aborted

close

message.destroy([error])

message.headers

message.httpVersion

message.method

message.rawHeaders

message.rawTrailers

message.setTimeout(msecs, callback)

message.statusCode

message.statusMessage

message.socket

message.trailers

message.url

http.METHODS

http 的方法列表

http.STATUS_CODES

http 状态码集合

http.createServer([requestListener])

返回一个新的http.Server实例

http.get(options[, callback])

返回一个新的http.ClientRequest实例

http.globalAgent

Agent全局实例，用于所有http 客户端请求。

http.request(options[, callback])

返回一个新的http.ClientRequest实例
options:

  1. protocol
  2. host
  3. hostname
  4. family
  5. port
  6. localAddress
  7. socketPath
  8. method
  9. path
  10. headers
  11. auth
  12. agent
  13. createConnection
  14. timeout
