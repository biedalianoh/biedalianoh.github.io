关于Js中运算与赋值的执行顺序

发表于 2017-01-11   |   分类于 Javascript

Js中涉及到赋值，调用，函数的执行，赋值，参数赋值等计算，赋值的机制并不是按照统一的顺序，在此对于常用的执行顺序进行总结。

参数的赋值

参数的解析顺序是： 从左到右
Es6提到TDZ(暂存死区)，指这样的程序区域：初始化前的变量或参数不能被访问。函数的参数既是这样的典型区域。

赋值表达式赋值

赋值顺序为：从右到左

运算符优先级

其中 () 优先级高于 . 高于 new
此处的 () 为把表达式括起来的括号，而不是 new 操作符后面跟随的 ()

var, function 声明提升

var 和 function 命令会提升至当前作用域顶端。var 命令只会提升定义，不会提升赋值。

let, const, class

let, const, class 不会进行声明提升。

import

import 会提升至作用域顶端。
