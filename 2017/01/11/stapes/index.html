Stapes.js 用法详解

发表于 2017-01-11   |   分类于 Javascript

Stapes.js是一个灵活，简单，轻量级的Javascript框架。因为框架的易用性以及所能提供的丰富的功能，在之前的项目中已经用到stapes.js，近期由于想查询一些问题，发现当时项目中没有相关的注释，加上官网api排列很乱，查阅起来很麻烦，便由此想到写一篇stapes.js入门，以增强记忆并以备之后查阅。

stapes下载

可在官网或cdn上下载。或使用npm，bower等加载

基本用法

一个stapes对象被创建时为常见的json格式，但是对象本身会被赋予构造函数，原型，并且在创建时，对象的原型链已经被创建。

当实例化时，构造函数constructor会被执行一次。其他函数被当作原型方法。

继承

继承是stapes中很核心的思想。stapes能提供多种方式的继承。并且其他常见框架的继承方式均被支持。

简单继承

subclass本身即为继承函数，Module = Stapes.subclass()即为Module继承Stapes。

由于继承时，构造函数constructor不会被继承，因此如果想继承构造函数，需要单独赋值。

extend继承

extend能更灵活的继承新的方法和属性。

也支持多重继承

对于参数继承也很有用

extend也能为Stapes扩展新的方法和属性，而且即便已经new过的stapes类，也能直接使用extend过的方法。

原型

给module添加方法和属性。功能与extend类似。

直接在subclass中给原型添加方法更简单，当然也可以在之后使用proto添加。添加的方法为原型方法，非静态方法。

proto也和extend一样支持多重添加。

事件和方法

on

监听Stapes事件，或module实例的事件。可设置单独监听，或多个监听。

其中，最后一个参数为指定this对象为谁。

同时，也可以监听module上的所有事件。

off

移除监听。

emit

触发事件。

get

通过key返回属性，如果没有key对应的属性，则返回null

getAll

获取module上的所有属性。这个方法返回的是module属性的clone

####getAllAsArray
获取module上所有属性值，并返回一个数组。这个方法返回的是module属性的clone

set

设置一个或多个属性。多个属性使用json作为参数。
每个属性的设置set方法，都会触发事件。如：
change—-均会触发
create—-module上没有这个属性时
update—-module上有这个属性时
remove—-删除属性时触发
eventType:key—-set时会触发相应的对应key的事件。如module.set(‘name’,’zhou’), 则change:name会被触发

当然，如果在set时不想触发任何事件，也可以设置。

当在使用set时，若想获取原本的值和设置之后的值，可以使用mutate来代替change。

需要注意，不论是change，update，mutate等方法，都是在module.set()时才会被触发，直接修改属性不会触发

set方法返回module，因此支持链式调用。
最后，new出来的Module类可以重写set方法，以适用项目所需。

mixinEvents

把stapes方法添加至不是stapes类型的object或者function上。

当然可以通过mixinEvents创建一个module

也可以把stapes方法添加给函数。

push

添加一个值或一个数组给module。每个值默认生成一个uuid作为对应的key。
同时也支持参数确认是否触发事件。

push返回m，因此支持链式调用。

remove

删除属性。会触发remove，remove:name和change，chang:name事件。

remove返回module，因此支持链式调用。

update

更新module的属性，通过function的返回值来实现。会触发change和update事件。

也可以直接传一个function，update会对module的每一个属性进行遍历操作。

第二个参数为指定this值，不传即为module本身。

filter

筛选器。通过函数筛选，循环每个module的属性，并返回筛选后符合条件的数组。

has

检查是否具有某个属性或方法。返回布尔值

map

循环module的每个属性，并对每个属性都进行操作之后，返回数组。

第二个参数为指定this指向，不填写则默认为当前module。

size

返回module的属性数量。
